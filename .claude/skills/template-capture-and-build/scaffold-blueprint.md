# Niche Template Scaffold Blueprint

What every niche template at `website-factory/templates/{niche-slug}/` must
contain. Module 2D Phase 6 follows this blueprint to scaffold the Vite + React
template alongside the niche playbook generated in Phase 5b.

## Source of truth

The baseline template at `website-factory/templates/website-template/` is the
reference implementation. Read it before scaffolding so version drift doesn't
break the factory's build tooling.

```bash
ls website-factory/templates/website-template/
cat website-factory/templates/website-template/package.json
cat website-factory/templates/website-template/src/config/brand-dna.example.js
cat website-factory/templates/website-template/PHOTO-MANIFEST.md
cat website-factory/templates/website-template/scripts/inject-theme.mjs
cat website-factory/templates/website-template/tailwind.config.js
cat website-factory/templates/website-template/src/index.css
```

The baseline ships with the foundation tokens (Phase C) already baked in:
- Modular type scale (display-sm/md/lg/xl named tokens)
- 8pt spacing (section-gap, section-gap-lg, card-pad)
- Layered shadows (shadow-card / card-lg / floating)
- Premium easings (ease-premium-out / in / in-out + spring-bounce)
- Variable fonts + opsz axis (Fraunces + Plus Jakarta Sans defaults)
- `prefers-reduced-motion: reduce` block
- Motion components: ScrollRevealHeadline, PageTransition
- framer-motion dependency

Every niche template inherits these unchanged. The niche playbook decides
which Tier 2 motion patterns to enable (`motion-preset.json`) and which font
families to load per-client (`brand-dna.typography`).

## File-by-file contract

### `package.json`

Inherits from website-template (`{niche-slug}-template` name, same scripts, same
dependencies including `framer-motion`). Mirror website-template's actual dependency
versions to avoid drift during the factory's build phase.

### `tailwind.config.js`

Copy website-template's verbatim, the foundation tokens (modular scale, 8pt
spacing, layered shadows, premium easings) are universal. The niche
playbook's `motion-preset.json` and `theme.json` modify behaviour at
runtime via brand-dna; the Tailwind config itself doesn't need niche
customisation.

### `src/index.css`

Copy website-template's verbatim. The foundation block (variable font @import,
`font-optical-sizing: auto`, `prefers-reduced-motion`, easings as CSS
variables, tabular-nums and prose-narrow utilities) ships unchanged.
`inject-theme.mjs` rewrites the `:root` palette and `@import` font URLs
per client.

### `src/config/brand-dna.defaults.js` (NEW)

Niche-stable defaults Module 2D writes. The per-client `brand-dna.js`
merges on top.


### `src/config/brand-dna.js`

Per-client config with `__REQUIRED__*__` sentinels. Stage 10.1 fills these
from research + intake. The shape mirrors `brand-dna.example.js` (the
existing reference). After the merge with `brand-dna.defaults.js`, no
sentinel should survive, `inject-theme.mjs` fails the build if it finds one.

### `src/App.jsx`

Wraps Routes in `<PageTransition>` (foundation component from Phase C). The
fade-and-translate transition fires on every route change.

### `src/components/{SectionName}.jsx` template

Pattern unchanged from the existing website-template components: every visible
string comes from `brandDNA.copy.*`, every image from `brandDNA.images.*`,
sentinels live only in `brand-dna.js`.

Niche-specific section components are scaffolded by Module 2D per the
spec. The component name reflects niche vocabulary (e.g. `RoomsSuites.jsx`
for hospitality, `Services.jsx` for contractor), Module 2D writes
`vocabulary.json` so the build agent can choose between alternates if needed.

### `PHOTO-MANIFEST.md`

Cross-references `niche-playbook/photo-manifest.json`. The markdown file
documents every image slot for human readability; the JSON file is the
machine-readable contract Stage 4 consumes.

### `scripts/inject-theme.mjs`

Copy website-template's verbatim. The script rewrites:
- `:root` palette CSS variables (from `brandDNA.palette`)
- Google Fonts `@import` (from `brandDNA.typography`)
- `<html data-theme-mode="...">` attribute (from `brandDNA.theme_mode`)
- JSON-LD schema block from `brandDNA.company` / `address` / `hours` /
  `reviews`. The schema.org `@type` field reads from `brandDNA.jsonLdType` if
  present (the niche playbook may set this in `brand-dna.defaults.js`),
  falling back to `LocalBusiness`.

### `index.html`

Title + meta description filled by `inject-theme.mjs` from `brandDNA.meta`.

## Validation contract

After scaffolding, the build must succeed and produce a `dist/` that:

1. Contains every `__REQUIRED__` sentinel string from `brand-dna.js` (proof
   placeholders survive into the per-client overlay).
2. Validates every niche-playbook JSON file against its schema at
   `website-factory/references/niche-playbook/schemas/`.
3. Passes `website-factory/tools/copy-lint.py --check --include-niche {slug}`
   on `niche-playbook/copywriting.md` and `niche-playbook/cro-rules.md`.
4. Lighthouse can analyze `dist/index.html` (run if available).

If any check fails, the scaffold is broken. Report and stop.

## What this blueprint does NOT enforce

- Pixel-perfect visual match to the inspiration screenshot. The scaffolded
  React has the right section order, the right component contract, the right
  brand-dna hooks, the right CSS variables. Visual finesse beyond that is
  the factory's design fidelity stage (Stage 10.4a) and the niche template
  iterates via `/refine-template` after the first client build.
- Production-grade animations. The scaffold includes hooks for animations
  the spec mentions; actual implementation lives in the factory's uplift
  stage or refinement loop.
- Specific photography quality. The factory's asset-scraper (Stage 4)
  handles per-client photo harvesting against the niche playbook's manifest.
